Product Case Study

Aryanony AI Bot

Project Summary

  • Challenge: Answering repeating user inquiries about rates, slots, and services manually limits 24/7 lead capture efficiency.
  • Solution: A WhatsApp and Telegram AI agent built on OpenAI Assistants API and orchestrated via n8n.
  • Outcome: Captures qualified lead scopes automatically, logs details in Notion, and triggers calendar booking workflows without human lag.

The messaging overhead

For independent consultants and technical PMs, handling repetitive messaging traffic on social handles or direct messaging apps (availability check, stack scope, fee terms) consumes hours of consulting bandwidth. If responses are delayed, potential clients often look elsewhere.

Building the AI integration

I built Aryanony AI Bot as a dedicated messaging agent. The bot is trained on my project blueprints, case study parameters, and rate models. When a user sends a query, it evaluates the scope, answers questions, gathers contact parameters, and provides a direct calendar booking slot.

Technical Stack & Flow:

  • Agent Framework: OpenAI Assistants API with specialized instructions and function calling capabilities.
  • Orchestrator: n8n self-hosted webhook nodes directing payloads.
  • Messaging Channels: WhatsApp Business Cloud API and Telegram Bot API.
  • Data Sync: Notion database logs recording conversation summaries, client requirements, and contact details.

Automation Outcomes

Aryanony AI Bot operates 24/7, providing instant answers to initial scoping inquiries. By qualifying leads automatically before they reach my calendar, it keeps my schedule filled with productive conversations.
